# PM Skill 用户配置指南

## Profile 文件

**文件位置：** `~/.pm-skill-profile.json`

Profile 用于存储用户偏好和上下文，让 skill 能提供更个性化的分析和建议。

---

## 首次使用流程

当检测到用户没有 Profile 时，触发 Profile 创建流程：

### Step 1: 欢迎与说明
```
欢迎使用 PM Skill 👋

为了给你提供更精准的分析和建议，我需要先了解一些你的背景信息。

这个过程只需要 1-2 分钟，配置完成后会记住你的偏好。

准备好了吗？回复 "开始" 继续。
```

### Step 2: 批量问答收集（3轮）

**【第二轮】工作环境（2个问题）**

**【第三轮】偏好设置（5个问题）**

## Profile 使用逻辑

### 读取 Profile
当 skill 被触发时：
1. 检查 `~/.pm-skill-profile.json` 是否存在
2. 如果不存在 → 触发 Profile 创建流程
3. 如果存在 → 读取并应用到当前分析中

### 应用 Profile
根据 Profile 中的偏好，调整输出：

| Profile 字段 | 影响 |
|-------------|------|
| `role` | 调整分析深度和术语使用 |
| `experience_years` | 控制案例复杂度、术语解释深度、建议颗粒度 |
| `industry` | 补充行业特定的 checklists 和案例 |
| `methodology` | 调整排期模板和流程建议 |
| `output_format` | 决定输出格式（Markdown/飞书/思维导图） |
| `detail_level` | 控制输出的详细程度 |
| `focus_area` | 针对特定领域的风险点和最佳实践 |
| `communication_style` | 调整语言风格和表达方式 |

### 更新 Profile
用户可以随时更新配置：
```
用户：/pm config
或：更新我的 PM skill 配置
```

进入 Profile 更新模式，重新回答所有或部分问题。

## 基于 Profile 的个性化示例

### 示例 1：AI 领域 Senior PM
```
用户背景：AI/大模型、大厂、业务线PM、关注AI产品

输入：帮我评估一个AI视频翻译功能

输出特点：
- 自动补充 AI 产品特有的风险点（模型幻觉、合规、算力成本）
- ROI 分析包含 GPU 成本、模型调用费用
- 风险评估强调 AI 伦理和监管合规
- 参考行业案例：HeyGen、Runway、Pika
```

### 示例 2：电商领域 PM Lead
```
用户背景：电商、小厂、独立负责、关注增长和商业化

输入：帮我拆解一个直播带货功能

输出特点：
- 执行方案包含电商特有的模块（商品、订单、支付、售后）
- ROI 分析聚焦 GMV、转化率、客单价
- 排期考虑电商大促时间节点
- 风险点强调供应链、库存、物流
```

### 示例 3：SaaS 领域 Senior PM
```
用户背景：企业服务、中厂、平台PM、关注基础体验

输入：帮我排期一个权限管理重构

输出特点：
- 强调企业级产品的安全性和稳定性
- 考虑多租户、数据隔离等技术方案
- 风险评估包含迁移成本、兼容性
- 输出格式偏向技术文档风格
```

## 实现说明

### Profile 存储
- 文件路径：`~/.pm-skill-profile.json`
- 格式：JSON
- 权限：用户可读写

### 首次检测逻辑
```
用户触发 /pm
    │
    ├─ 检查 ~/.pm-skill-profile.json
    │   ├─ 存在 → 读取并应用 → 继续分析
    │   └─ 不存在 → 触发 Profile 创建流程
    │       └─ 完成创建后 → 保存 → 继续分析
    │
    └─ 分析时动态应用 Profile 偏好
```

### 向后兼容
- 如果用户拒绝创建 Profile，skill 仍可使用，但使用默认配置
- 默认配置：`{"detail_level": "标准", "output_format": "Markdown"}`
