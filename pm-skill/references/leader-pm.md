# PM Leader 宏观视角指南

## 核心职责
从 PM Leader 的角度和宏观视角，思考事项的逻辑、ROI 和可执行性，梳理风险点和需要 check 的地方。

## 分析框架

### 0. 背景信息收集阶段 ⭐关键
**目标**: 在判断"值不值得做"之前，先充分了解背景和上下文

**原则**: 没有足够信息，就无法做出准确判断；宁可多问，不可瞎猜

**信息收集清单模板**:
```
根据用户初步描述，列出需要了解的背景信息：

【关于需求背景】
- 这个需求的来源是什么？（用户反馈？老板要求？竞品跟进？）
- 解决什么核心问题？痛点的严重程度？
- 不做会有什么后果？

【关于业务现状】
- 你们现在的业务数据怎么样？（DAU/GMV/留存等）
- 是否有类似的项目尝试过？结果如何？
- 公司/部门当前的战略重点是什么？

【关于约束条件】
- 有硬性时间要求吗？（如必须Q3上线）
- 预算上限是多少？
- 有哪些不能做/不能碰的限制？（技术/合规/政治）

【关于竞争环境】
- 竞品在做吗？做得怎么样？
- 我们的差异化优势是什么？
```

**执行步骤**:
1. 列出背景信息收集清单（5-8个问题）
2. 向用户提问："为了准确评估这个项目的 ROI 和可行性，我需要先了解一些背景信息..."
3. 等待用户回答
4. 确认理解无误后，再进入分析阶段

**警示**: 如果背景信息不清楚，ROI 分析和风险评估都会失真！

---

### 1. 战略契合度评估
**目标**: 判断这件事是否值得做，是否契合当前战略

**评估维度**:

#### 1.1 业务价值
| 问题 | 评估要点 |
|------|----------|
| 解决什么核心问题？ | 是否是用户/业务的真痛点？还是伪需求？ |
| 目标用户是谁？ | 用户群体有多大？价值高吗？ |
| 与现有业务的关系？ | 是独立新业务？还是现有业务的增强？ |
| 竞争优势是什么？ | 为什么是我们做？护城河在哪里？ |

#### 1.2 战略契合
- [ ] 是否契合公司当前战略方向？
- [ ] 是否契合部门 OKR/目标？
- [ ] 是进攻型项目（开拓新市场）还是防守型项目（巩固现有）？
- [ ] 不做会有什么后果？

**输出物**:
- 业务价值陈述（1页纸）
- 战略契合度评估结论

---

### 2. ROI 分析
**目标**: 评估投入产出比，判断是否划算

#### 2.1 成本估算
```
成本构成
├── 一次性成本
│   ├── 人力成本: [人数] × [月数] × [人均成本]
│   ├── 技术成本: 服务器、存储、带宽
│   ├── 第三方服务: API费用、SDK费用
│   └── 其他: 合规、法务、专利
└── 持续成本
    ├── 运维成本: 每月固定支出
    ├── 内容成本: 内容生产/采购费用
    └── 推广成本: 获客成本 × 目标用户数
```

#### 2.2 收益估算
```
收益类型
├── 直接收益
│   ├── 收入增长: 预计GMV/收入增长
│   ├── 成本节约: 效率提升带来的成本节约
│   └── 用户付费: ARPU × 用户数
├── 间接收益
│   ├── 用户增长: DAU/MAU提升
│   ├── 留存提升: 留存率改善
│   ├── 品牌提升: 品牌价值、口碑
│   └── 数据资产: 积累的数据价值
└── 战略收益
    ├── 市场卡位: 抢占先机
    ├── 能力积累: 团队能力提升
    └── 生态建设: 完善产品矩阵
```

#### 2.3 ROI 计算
```
短期 ROI (6个月):  [短期收益] / [短期投入]
中期 ROI (1年):   [中期收益] / [总投入]
长期 ROI (3年):   [长期收益] / [总投入]

回收周期: 预计多少个月回本
```

**决策参考**:
| ROI 情况 | 建议 |
|----------|------|
| 短期 ROI > 1 | 优先做，快速验证 |
| 中期 ROI > 1 | 值得投入，安排资源 |
| 长期 ROI > 1，短期 < 1 | 战略投入，需要长期承诺 |
| 各期 ROI 都不理想 | 重新评估或放弃 |

**输出物**:
- 成本收益分析表
- ROI 计算结论
- 投资决策建议

---

### 3. 可执行性评估
**目标**: 判断这件事能不能做成

#### 3.1 技术可行性
- [ ] 技术方案是否成熟？是否有先例？
- [ ] 核心技术风险是什么？有备选方案吗？
- [ ] 技术团队有能力实现吗？
- [ ] 第三方依赖可靠吗？有 Plan B 吗？

#### 3.2 资源可行性
- [ ] 人力是否充足？核心人员能否到位？
- [ ] 预算是否足够？超支风险如何？
- [ ] 时间窗口是否合适？错过会怎样？

#### 3.3 组织可行性
- [ ] 需要跨部门协作吗？各方意愿如何？
- [ ] 决策链路清晰吗？关键决策人支持吗？
- [ ] 是否存在政治/利益冲突？

#### 3.4 市场可行性
- [ ] 市场时机是否合适？
- [ ] 监管政策有影响吗？
- [ ] 竞品动态如何？我们有差异化优势吗？

**输出物**:
- 可行性评估矩阵
- 关键阻碍清单
- 推进建议

---

### 4. 风险识别与应对
**目标**: 识别潜在风险，制定应对策略

#### 4.1 风险分类框架
```
风险类型
├── 需求风险
│   ├── 需求理解偏差
│   ├── 需求变更频繁
│   └── 用户不买账
├── 技术风险
│   ├── 技术方案不可行
│   ├── 性能不达标
│   ├── 第三方服务故障
│   └── 安全漏洞
├── 进度风险
│   ├── 排期过于乐观
│   ├── 关键人员离职
│   └── 依赖方延期
├── 商业风险
│   ├── 竞品抢先上线
│   ├── 政策变化
│   ├── 市场环境变化
│   └── 合作方违约
└── 运营风险
    ├── 内容供应不足
    ├── 用户增长不及预期
    └── 运营成本超支
```

#### 4.2 风险评估矩阵
| 风险项 | 可能性 | 影响程度 | 风险等级 | 应对策略 | 负责人 |
|--------|--------|----------|----------|----------|--------|
| 示例：第三方API不稳定 | 中 | 高 | 🔴 高 | 1. 签SLA协议<br>2. 准备降级方案<br>3. 备选供应商 | 张三 |

**风险等级定义**:
- 🔴 高风险: 必须解决或制定详细应对计划
- 🟡 中风险: 需要监控，准备应对措施
- 🟢 低风险: 接受并持续观察

**输出物**:
- 风险登记册
- 风险应对计划
- 应急预案

---

### 5. 关键 Checkpoints
**目标**: 确保关键节点有检查机制

#### 5.1 决策检查点
| 阶段 | 检查问题 | Go/No-Go 标准 |
|------|----------|---------------|
| 立项前 | 战略契合吗？ROI合理吗？ | 战略契合度>7分，预期ROI>1 |
| 需求评审 | 需求清晰吗？范围确定吗？ | 需求文档完整，范围明确 |
| 方案评审 | 方案可行吗？风险可控吗？ | 技术方案通过，风险评估完成 |
| 上线前 | 质量达标吗？运营准备好了吗？ | 测试通过，运营准备就绪 |
| 上线后 | 数据符合预期吗？ | 核心指标达到预期的80% |

#### 5.2 持续监控指标
```
健康度仪表盘
├── 进度健康度
│   ├── 里程碑按时完成率
│   └── Bug收敛趋势
├── 质量健康度
│   ├── 测试覆盖率
│   └── 线上故障数
├── 团队健康度
│   ├── 人员稳定性
│   └── 团队满意度
└── 业务健康度
    ├── 核心指标达成率
    └── 用户反馈 sentiment
```

**输出物**:
- Checkpoint 检查清单
- 健康度仪表盘设计

## ⚠️ 致命陷阱警示

### 陷阱：没有背景就做判断
| 表现 | 后果 | 应对 |
|------|------|------|
| 不了解业务现状就评估 ROI | 成本/收益估算严重失真 | **强制背景信息收集阶段** |
| 不知道公司战略就谈契合度 | 战略契合度评估错误 | **先问公司/部门当前重点** |
| 不清楚历史尝试就下结论 | 重复踩坑，浪费资源 | **先问是否尝试过，结果如何** |

> **铁律**: 背景信息不清楚，所有分析都是瞎猜！

## Leader 决策 checklist

### 立项前必问
- [ ] 这个需求从哪来的？用户真实反馈还是老板拍脑袋？
- [ ] 解决的是第几优先级的问题？
- [ ] 如果不做，最坏的结果是什么？
- [ ] 做了之后，成功是什么样子？怎么衡量？
- [ ] 团队现在有资源做吗？
- [ ] 竞品在做吗？我们落后多少？

### 进行中必问
- [ ] 进度是否符合预期？偏差在哪里？
- [ ] 有没有新的风险出现？
- [ ] 需求变更了吗？影响多大？
- [ ] 团队状态怎么样？有人要离职吗？
- [ ] 还需要我协调什么资源？

### 上线后必问
- [ ] 数据符合预期吗？差多少？
- [ ] 用户反馈怎么样？
- [ ] 有什么意料之外的问题？
- [ ] 下一步迭代方向是什么？
